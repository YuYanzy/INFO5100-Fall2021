Problem 3

Type coercion means that when two different types of values are calculated together, one type will be automatically and forcibly converted to the other type so that the operation can be performed reasonably.

Example 1: "1" + 20, result is "120".
In this example, a string and a number are added which is not allowed to be computed in other languages like Python because string can only concatenate to string.
To rationalize this operation, the number 20 is forcibly converted to the string type, "20", and then contact with the former string "1" to get the result "120".

Example 2: [1, 2, 0] + 20, result is "1, 2, 020".
In this example, an array and a number are added which is not allowed to be computed in other languages like Python because list can only concatenate to list.
To rationalize this operation, the array and the number are forcibly converted to the string type, and then they are concatenated to get the result "1, 2, 020".
